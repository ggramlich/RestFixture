!3 GET

!**** Notes
A GET request is performed on the resource whose URI is specified as an argument to the ''get'' method.
The following boolean methods verify:
 * ''has status'' the expected return HTTP code,
 * ''has header'' or ''has headers'' a subset of headers - regular expressions are allowed
 * ''has body'' a list of XPaths to be executed in the body. Each XPath is run and it suceedes only if the result of the execution returns at least one node
****!

-!|script|
|GET|/resources/100|has status|200|has body  |!-/resource/type[text()='large content']-!|
|GET|/resources/0  |has status|200|has header|Content-Length : [\d]+      |has body     |!-
 /resource/name[text()='a funky name'] <br/>
 /resource/data[text()='an important message']
-!|


A GET request properly handles query strings. In this example the stub resource echoes the query string in a Query-String header

-!|script|
|GET|/resources/0?param=echome|has status|200|has header|!-Content-Length : [\d]+
Query-String : param=echome -!|has body|!-
 /resource/name[text()='a funky name'] <br/>
 /resource/data[text()='an important message']
-!|

Get a non existing resource should not find it

!**** Note
Instead of XPaths you can specify 'no-body' if the operation is expected to return an empty body
****!

-!|script|
|GET|/resources/20     |has status|404|has header|Content-Length : 0|has body|no-body|
|GET|/resources/20.json|has status|404|has header|Content-Length : 0|has body|no-body|

!**** XPaths and namespaces
Namespaces are supported by declaring the alias/uri map in the RestFixtureConfig, using the key restfixture.xml.namespace.context (See SuiteSetUp)
****!

-!|script|
|GET|/resources/0|has status|200|has header|Content-Length : [\d]+|has body|!-
 /resource/nstag/ns1alias:number[text()='3']
-!|

!**** Json
Expectations can be asserted also in json format for responses whose content type is 'application/json' or 'application/x-javascript'
****!

-!|script|
|GET|/resources/1.json|has status|200|has header|Content-Type : application/json;charset=ISO-8859-1|has body|!-
 /resource/name[text()='a funky name'] <br/>
 /resource/data[text()='an important message']
-!|
